# 段永平投资 Skill (Duan Yongping Investing Skill)

> 把段永平的投资方法论蒸馏成一套可在 Claude / Cursor / Codex / 任意 LLM 平台执行的判断流程。

## 这是什么

一个 **AI Skill**——把段永平公开发言、经典案例(网易、苹果、茅台、PDD)中的方法论,整理成一套**可执行的7步决策流程**。

加载这个 skill 后,你的 AI 助手就能用段永平的视角帮你分析公司、判断标的、做投资决策时提醒你卡在哪一步。

## 跟同类项目的区别

市面上已经有把段永平问答录拆成多个独立 skill 的项目(比如 `kangarooking/duan-yongping-skill` 把书拆成 15 个零散 skill)。**这个项目的差异在于**:

| 维度 | 同类项目 | 本项目 |
|---|---|---|
| 组织方式 | 知识点平铺(15 个独立 skill) | 决策流程(1 个完整流程,7 步 + 支撑文档) |
| 使用场景 | 学习 / 查询知识点 | 实际做投资判断 |
| 输出形式 | 单点回答 | 结构化分析 + 建议仓位 + 诚实提醒 |
| 边界处理 | 不区分标准/特殊 case | 明确区分标准方法论(苹果/茅台)和特殊豁免(PDD) |
| 跨平台 | Claude Skill 格式 | Claude Skill 格式 + 平台无关使用指南 |

**简单说**:那个项目教你"段永平说过什么",这个项目教你"段永平面对一只股票时会怎么做"。

## 为什么做这个

段永平方法论的精髓不是知识,是**判断流程**。读完他所有公开发言,你也未必能在面对一只新股票时做对决策——因为知识点的堆砌和实际决策流程的训练是两件事。

这个 skill 的设计目标:**当你打算买/卖一只股票时,它能强迫你回答正确的问题,在卡住的那一步停下来**。

## 怎么用

### 在 Claude 上(claude.ai 或 Claude Desktop)

1. 进入 Claude Settings → Capabilities → Skills
2. 上传整个 `duan-yongping-skill` 目录(打包成 .skill 文件或上传整个文件夹)
3. 在对话中直接问"用段永平方法论分析一下 X 公司"——skill 会自动触发

### 在 Cursor / Codex / Cline 上

这些工具支持自定义指令或 rules 文件:

1. 把 `SKILL.md` 的内容复制到你的 `.cursorrules` 或对应的 rules/instructions 文件里
2. 或者把整个项目目录放在工作区,在对话中说"参考 `duan-yongping-skill/SKILL.md` 分析一下..."
3. 必要时让 AI 调用具体的 reference 文件(比如 `references/case-moutai-2013.md`)

### 在 ChatGPT / Gemini / DeepSeek 等

这些平台不直接支持 skill,但可以用以下方式:

**方法 1:作为系统提示**

打开一个新对话,把 `SKILL.md` 全文作为系统提示(或 GPTs 的 instructions)粘贴进去。然后正常提问即可。

**方法 2:作为知识库**

如果平台支持自定义 GPTs / Bots,把整个项目目录作为知识库上传。AI 会按需查阅 references 和 examples。

**方法 3:手动触发**

在对话开头说:"请按照以下方法论分析问题:[粘贴 SKILL.md 全文]"——然后正常提问。

### 在任意 LLM 平台上的通用方法

把 `SKILL.md` 全文作为系统提示或对话的第一条消息,然后让 AI 在分析具体公司时:
- 先按 7 步流程逐项分析
- 在某一步卡住时,提醒你卡在哪
- 输出结构化结论和诚实提醒

## 核心方法论速览

完整流程见 `SKILL.md`,这里只列骨架:

### 7 步决策流程

1. **能用一句话讲清楚生意吗?** — 讲不清就放弃
2. **你或身边有人是用户吗?** — 没用户视角,判断质量打折
3. **反向证明:为什么别人做不到?** — 没有反向答案=没有护城河
4. **管理层"本分"吗?** — 听其言、观其行、察其果
5. **10年后还在吗?** — 不能确认就放弃
6. **估值便宜吗?** — 用"扣现金后EV/E + 模糊正确"判断,15%隐含回报为基准
7. **仓位与持有期** — 集中持仓,默认永远持有

### 三个元原则

- **做对的事 + 把事做对**
- **不为清单 > to-do清单**
- **延迟满足 + 时间是朋友**

### 三个特殊case(进阶)

- **管理层认知 > 生意认知**(PDD豁免) — 严格条件下成立
- **危机中的独立验证**(茅台塑化剂) — 关键变量花成本验证
- **Category 重新分类**(苹果消费品定位) — 真正的alpha来源

## 这个 skill 不能做什么

诚实地说出它的边界:

- ❌ **不预测股价** — 任何"会涨到多少"、"什么时候反弹"的问题它都拒绝
- ❌ **不替你决策** — 它的输出是分析,不是命令
- ❌ **不适合所有行业** — 对周期股、技术快速迭代行业、早期初创公司,它会倾向拒绝(这是方法论的特征,不是缺陷)
- ❌ **不能模仿段永平的所有持仓** — 段永平自己都说"我不推荐别人买拼多多";复制持仓不是段永平方法论
- ❌ **不能取代你自己的研究** — 它强迫你回答正确的问题,但答案要你自己提供

## 测试用例

试试这些问题,验证 skill 是否正常工作:

1. "用段永平方法论分析一下贵州茅台,假设股价 1500 元"
2. "段永平会怎么看英伟达?"
3. "我应该跟着段永平买苹果吗?"(预期回答:不应该简单跟随,要建立自己的安全边际)
4. "PDD 现在值得买吗?段永平不是重仓了吗?"(预期回答:解释PDD是特殊case,不能简单复制)
5. "帮我分析一只小盘消费品股票"(让 AI 走完完整流程)
